- **Quiz schema fix:** Changed option-ID validation from `uuidSchema` to `z.string().min(1).max(64)` — option IDs are arbitrary strings by design (`"a"`, `"b"`, `"true"`, `"false"`), not UUIDs. Question-ID keys kept as `uuidSchema` (those ARE UUIDs).

**Lessons Learned:**
- **Silent validation failures are catastrophic.** Zod schema mismatch (`uuidSchema` vs actual string IDs) caused `validateAnswerForQuestion` to return `{ok: false}` for every answer — scoring every quiz 0. No error was surfaced to users. Every paying user was locked out of gated lessons behind quizzes. Always verify schema constraints match actual data, especially after DB seeding/migration.

- **DB audit before schema assumptions.** The initial report was "True/False broken" — a DB-wide audit (`WHERE NOT (id ~ uuid_regex)`) revealed the scope was 100% of questions, not just one type.

**Context:** Fixing broken webinar signup flow on underdogsales.com before ad launch

**Key Exchanges:**
- User reported webinar signup stopped working after previous session's changes
- Debugging revealed 4 stacked bugs causing the failure, each masked by the next

**Decisions Made:**
- Switched from Brevo `templateId` to inline HTML for Email 1 (transactional send) — Brevo's `{{ contact.* }}` merge tags only resolve in campaign sends, not transactional `/smtp/email` calls. Emails 2–10 (campaigns) still use templates fine.
- Used JWT-format anon key instead of new `sb_publishable_*` format — Supabase Edge Functions don't support the new key format yet
- Replaced `support@underdogsales.com` with `gsc@underdogsales.com` across all user-facing copy
- Excluded sprint popup from `/webinar/*` paths to avoid distracting from RSVP CTA

**Lessons Learned:**
- **Brevo transactional vs campaign sends are fundamentally different:** `templateId` with merge tags (`{{ contact.FIELD }}`) only works in campaign sends. Transactional `/smtp/email` requires explicit `sender`, `subject`, and `htmlContent` — even when a `templateId` is provided, merge tags won't resolve.
- **`NEXT_PUBLIC_*` env vars are baked at build time on Vercel:** `vercel redeploy` (which reuses the build artifact) won't pick up new env var values. A fresh `vercel --prod` build is required.
- **Supabase Edge Functions still require JWT anon key:** The newer `sb_publishable_*` key format is not accepted by Edge Functions — must use the classic JWT-format `eyJ...` anon key.
- **Stacked bugs mask each other:** The env var issue hid the key format issue, which hid the Brevo template issue. Each fix revealed the next layer.
